# Webデザインチェック スキル

このスキルは、Webサイトの品質を総合的にチェックするためのガイドラインです。リリース前やレビュー時に使用してください。

---

## 📋 クイックチェックリスト（概要）

```
□ デザイン一貫性    - 色、フォント、スペーシングの統一
□ レスポンシブ     - モバイル、タブレット、デスクトップ対応
□ アクセシビリティ  - 誰でも使えるか
□ パフォーマンス   - 読み込み速度
□ SEO / LLMO      - 検索エンジン・AI対策
□ ユーザビリティ   - 使いやすさ
□ クロスブラウザ   - 主要ブラウザ対応
□ コード品質      - 保守性・可読性
```

## 🎨 1. デザイン一貫性チェック

### カラーパレット

| チェック項目 | 確認方法 |
|------------|---------|
| □ プライマリカラーが統一されているか | 全ページでブランドカラーが同じか確認 |
| □ セカンダリカラーが適切に使われているか | アクセントとして一貫して使用されているか |
| □ テキストカラーのコントラスト比が4.5:1以上か | [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) |
| □ ホバー・アクティブ状態の色が定義されているか | インタラクティブ要素をすべて確認 |
| □ エラー・成功・警告の色が統一されているか | フォームやアラートを確認 |

```css
/* 推奨：CSS変数でカラーを一元管理 */
:root {
  --color-primary: #6366f1;
  --color-secondary: #8b5cf6;
  --color-accent: #ec4899;
  --color-success: #22c55e;
  --color-warning: #f59e0b;
  --color-error: #ef4444;
  --color-text: #1f2937;
  --color-text-muted: #6b7280;
  --color-bg: #ffffff;
  --color-bg-alt: #f9fafb;
}
```

### タイポグラフィ

| チェック項目 | 確認方法 |
|------------|---------|
| □ フォントファミリーが統一されているか | 見出し/本文で一貫性があるか |
| □ フォントサイズの階層が明確か | h1 > h2 > h3 > p の順になっているか |
| □ 行間（line-height）が適切か | 本文は1.5〜1.8が目安 |
| □ 文字間（letter-spacing）が調整されているか | 見出しは少し広め |
| □ 1行の文字数が適切か | 45〜75文字（日本語は25〜40文字）が目安 |

```css
/* 推奨：フォントサイズのスケール */
:root {
  --font-size-xs: 0.75rem;   /* 12px */
  --font-size-sm: 0.875rem;  /* 14px */
  --font-size-base: 1rem;    /* 16px */
  --font-size-lg: 1.125rem;  /* 18px */
  --font-size-xl: 1.25rem;   /* 20px */
  --font-size-2xl: 1.5rem;   /* 24px */
  --font-size-3xl: 1.875rem; /* 30px */
  --font-size-4xl: 2.25rem;  /* 36px */
  --font-size-5xl: 3rem;     /* 48px */
}
```

### スペーシング

| チェック項目 | 確認方法 |
|------------|---------|
| □ 余白のスケールが統一されているか | 4px, 8px, 16px, 24px, 32px, 48px等 |
| □ セクション間の余白が一貫しているか | 全セクションで同じ余白か |
| □ カード・コンポーネント内の余白が統一か | paddingが揃っているか |
| □ 要素間のgapが統一されているか | グリッドやフレックスで一貫性があるか |

```css
/* 推奨：スペーシングのスケール（8pxベース） */
:root {
  --space-1: 0.25rem;  /* 4px */
  --space-2: 0.5rem;   /* 8px */
  --space-3: 0.75rem;  /* 12px */
  --space-4: 1rem;     /* 16px */
  --space-5: 1.25rem;  /* 20px */
  --space-6: 1.5rem;   /* 24px */
  --space-8: 2rem;     /* 32px */
  --space-10: 2.5rem;  /* 40px */
  --space-12: 3rem;    /* 48px */
  --space-16: 4rem;    /* 64px */
  --space-20: 5rem;    /* 80px */
}
```

### コンポーネント

| チェック項目 | 確認方法 |
|------------|---------|
| □ ボタンのスタイルが統一されているか | primary, secondary, outline等 |
| □ カードのデザインが一貫しているか | 影、角丸、余白 |
| □ フォーム要素のスタイルが統一されているか | input, select, textarea |
| □ アイコンのサイズ・スタイルが統一されているか | 同じアイコンセットを使用 |
| □ border-radiusが統一されているか | 4px, 8px, 16px等で揃える |

## 📱 2. レスポンシブチェック

### ブレイクポイント確認

| 画面サイズ | 幅 | チェック |
|-----------|-----|---------|
| □ モバイル（縦） | 320px - 479px | 最小幅で崩れないか |
| □ モバイル（横） | 480px - 767px | |
| □ タブレット | 768px - 1023px | |
| □ デスクトップ | 1024px - 1279px | |
| □ 大型デスクトップ | 1280px以上 | |

### 各画面サイズでの確認項目

| チェック項目 | 確認方法 |
|------------|---------|
| □ 横スクロールが発生していないか | 各サイズで確認 |
| □ テキストが読みやすいサイズか | 最小14px以上 |
| □ ボタン・リンクがタップしやすいか | 最小44x44px |
| □ 画像がはみ出していないか | max-width: 100% |
| □ ナビゲーションが適切に変化するか | ハンバーガーメニュー等 |
| □ テーブルが横スクロール対応か | overflow-x: auto |
| □ モーダル・ポップアップが画面内に収まるか | 全画面表示等 |

### テスト方法

```bash
# Chrome DevToolsでのテスト手順
1. F12でDevToolsを開く
2. Ctrl+Shift+M でデバイスモード切り替え
3. 「Responsive」を選択してドラッグでサイズ変更
4. 各デバイスプリセット（iPhone, iPad等）でも確認
```

## ♿ 3. アクセシビリティチェック

### 画像

| チェック項目 | 確認方法 |
|------------|---------|
| □ すべての画像にalt属性があるか | `<img alt="">` |
| □ 装飾画像はalt=""（空）になっているか | 意味のない画像 |
| □ 複雑な画像に説明があるか | 図表やインフォグラフィック |

### フォーム

| チェック項目 | 確認方法 |
|------------|---------|
| □ すべてのinputにlabelが紐づいているか | `for`と`id`の対応 |
| □ 必須項目が明示されているか | `required`, `aria-required` |
| □ エラーメッセージが関連付けられているか | `aria-describedby` |
| □ プレースホルダーだけに頼っていないか | label必須 |

### キーボード操作

| チェック項目 | 確認方法 |
|------------|---------|
| □ Tabキーですべての要素にアクセスできるか | 順番にフォーカスが移動 |
| □ フォーカスインジケーターが見えるか | アウトラインを消していないか |
| □ Enterキーでボタン・リンクが動作するか | |
| □ Escキーでモーダルが閉じるか | |
| □ フォーカストラップが適切か | モーダル内に閉じ込め |

### ARIA

| チェック項目 | 確認方法 |
|------------|---------|
| □ ランドマークが設定されているか | header, nav, main, footer |
| □ aria-labelが適切に設定されているか | ナビゲーション等 |
| □ aria-expandedがトグルに設定されているか | メニュー開閉 |
| □ aria-currentが現在位置に設定されているか | ナビゲーション |
| □ aria-hiddenが装飾要素に設定されているか | アイコン等 |

### 自動チェックツール

```bash
# Lighthouseでのチェック
1. Chrome DevTools > Lighthouse タブ
2. 「Accessibility」にチェック
3. 「Analyze page load」をクリック

# axe DevTools（Chrome拡張）
1. 拡張機能をインストール
2. DevTools > axe DevTools タブ
3. 「Scan ALL of my page」
```

## ⚡ 4. パフォーマンスチェック

### Core Web Vitals

| 指標 | 目標値 | 確認方法 |
|-----|-------|---------|
| □ LCP (Largest Contentful Paint) | < 2.5秒 | 最大コンテンツの表示時間 |
| □ FID (First Input Delay) | < 100ms | 初回操作の応答時間 |
| □ CLS (Cumulative Layout Shift) | < 0.1 | レイアウトのずれ |
| □ TTFB (Time to First Byte) | < 800ms | サーバー応答時間 |

### 画像最適化

| チェック項目 | 確認方法 |
|------------|---------|
| □ 画像フォーマットが最適か | WebP/AVIF使用 |
| □ 画像サイズが適切か | 表示サイズ以上に大きくない |
| □ 遅延読み込みが設定されているか | `loading="lazy"` |
| □ 幅と高さが指定されているか | `width`, `height` でCLS対策 |
| □ 適切な圧縮がされているか | 品質80%程度 |

### リソース最適化

| チェック項目 | 確認方法 |
|------------|---------|
| □ CSSが圧縮されているか | 本番環境でminify |
| □ JSが圧縮されているか | 本番環境でminify |
| □ 不要なCSSが削除されているか | PurgeCSS等 |
| □ フォントが最適化されているか | woff2, font-display: swap |
| □ HttpやGzipが有効か | サーバー設定 |

### 読み込み最適化

| チェック項目 | 確認方法 |
|------------|---------|
| □ クリティカルCSSがインライン化されているか | ファーストビュー用 |
| □ 非同期読み込みが設定されているか | `async`, `defer` |
| □ preload/preconnectが設定されているか | 重要リソース |
| □ 外部リソースが最小限か | CDN依存を減らす |

### テストツール

| ツール | URL |
|-------|-----|
| PageSpeed Insights | https://pagespeed.web.dev/ |
| GTmetrix | https://gtmetrix.com/ |
| WebPageTest | https://webpagetest.org/ |
| Lighthouse | Chrome DevTools内蔵 |

## 🔍 5. SEO / LLMOチェック

### メタ情報

| チェック項目 | 確認方法 |
|------------|---------|
| □ titleが設定されているか | 30-60文字 |
| □ meta descriptionが設定されているか | 120-160文字 |
| □ canonical URLが設定されているか | 重複対策 |
| □ OGP（Open Graph）が設定されているか | SNS共有用 |
| □ Twitter Cardが設定されているか | Twitter用 |
| □ faviconが設定されているか | 各サイズ |

### 構造化データ

| チェック項目 | 確認方法 |
|------------|---------|
| □ JSON-LDが設定されているか | schema.org準拠 |
| □ Person/Organizationスキーマがあるか | サイト所有者情報 |
| □ WebSiteスキーマがあるか | サイト情報 |
| □ BreadcrumbListがあるか | パンくずリスト |
| □ FAQPageがあるか（該当する場合） | よくある質問 |

### HTMLのセマンティクス

| チェック項目 | 確認方法 |
|------------|---------|
| □ h1が1ページに1つだけか | 見出し階層 |
| □ 見出しの階層が正しいか | h1→h2→h3 |
| □ セマンティックタグを使用しているか | header, main, footer, article, section |
| □ 画像に適切なaltがあるか | SEO対策 |

### ファイル

| チェック項目 | 確認方法 |
|------------|---------|
| □ robots.txtがあるか | クローラー制御 |
| □ sitemap.xmlがあるか | サイトマップ |
| □ AIクローラーを許可しているか | GPTBot等 |

### テストツール

| ツール | URL |
|-------|-----|
| Rich Results Test | https://search.google.com/test/rich-results |
| Schema Markup Validator | https://validator.schema.org/ |
| Meta Tags Preview | https://metatags.io/ |

## 🖱️ 6. ユーザビリティチェック

### ナビゲーション

| チェック項目 | 確認方法 |
|------------|---------|
| □ 現在位置が分かるか | ハイライト、パンくず |
| □ 3クリック以内で目的に到達できるか | 主要コンテンツへ |
| □ ロゴがホームにリンクしているか | 慣例 |
| □ 404ページがあるか | エラーハンドリング |
| □ 戻るボタンが機能するか | ブラウザ履歴 |

### インタラクション

| チェック項目 | 確認方法 |
|------------|---------|
| □ ホバー状態が分かりやすいか | 色変化やアニメーション |
| □ クリック可能な要素が分かるか | カーソル変化 |
| □ ローディング状態があるか | 待機中のフィードバック |
| □ エラー状態が分かりやすいか | 赤色、アイコン等 |
| □ 成功状態が分かりやすいか | 緑色、チェックマーク |

### フォーム

| チェック項目 | 確認方法 |
|------------|---------|
| □ エラーメッセージが具体的か | 「入力してください」→「メールアドレスを入力してください」 |
| □ リアルタイムバリデーションがあるか | 入力中にフィードバック |
| □ 送信後のフィードバックがあるか | 完了メッセージ |
| □ 入力内容が保持されるか | エラー後 |

### コンテンツ

| チェック項目 | 確認方法 |
|------------|---------|
| □ テキストが読みやすいか | コントラスト、サイズ |
| □ リンクテキストが意味を持つか | 「こちら」ではなく具体的に |
| □ 外部リンクが分かるか | アイコンや新規タブ |
| □ 重要な情報が目立つか | 視覚的な強調 |

## ✅ 最終チェックリスト

### リリース前必須

```
□ 全ブレイクポイントで表示確認
□ 主要ブラウザで動作確認
□ Lighthouse スコア 90点以上
□ アクセシビリティツールでエラー0
□ コンソールエラーなし
□ リンク切れなし
□ 誤字脱字なし
□ 本番URLでOGP確認
□ robots.txt / sitemap.xml 確認
□ SSL証明書有効
```

### 確認コマンド

```bash
# Lighthouseをコマンドラインで実行
npx lighthouse https://example.com --output html --output-path ./report.html

# 画像最適化チェック
npx @squoosh/cli --help

# リンク切れチェック
npx broken-link-checker https://example.com
```
